# Redis Reader

Java readers for the Redis datasets generated by `../str.py` and `../ts.py`.

The project contains two entry points:

- `com.redis.equities.RedisReader` reads binary string values written by `str.py`.
- `com.redis.equities.TsReader` reads RedisTimeSeries series written by `ts.py`.

Both readers generate the expected key list locally, so they do not use `SCAN`.

## Requirements

- Java 17
- Maven
- Redis running on `localhost:6379`
- `redis-cli`
- RedisTimeSeries module when using `ts.py` and `TsReader`

## Build

Run from the `redis-reader` directory:

```bash
mvn clean package
```

This creates a shaded jar:

```bash
target/redis-reader-1.0-SNAPSHOT.jar
```

## Load String Data and Run `RedisReader`

From the `redis-reader` directory:

```bash
python3 ../str.py | redis-cli --pipe

java -Dredis.host=localhost -Dredis.port=6379 \
    -Dreader.threads=1 \
    -Dreader.pipelineBatch=500 \
    -Dreader.printPoints=false \
    -DMAX_KEYS=0 \
    -cp target/redis-reader-1.0-SNAPSHOT.jar \
    com.redis.equities.RedisReader
```

`str.py` writes each key as a Redis string using `APPEND`. Each value contains
5040 binary records. Each record is 12 bytes:

- little-endian `int64` timestamp in milliseconds
- little-endian `float32` value

## Load TimeSeries Data and Run `TsReader`

From the `redis-reader` directory:

```bash
python3 ../ts.py | redis-cli --pipe

java -Dredis.host=localhost -Dredis.port=6379 \
    -Dreader.threads=1 \
    -Dreader.pipelineBatch=500 \
    -Dreader.printPoints=false \
    -DMAX_KEYS=0 \
    -cp target/redis-reader-1.0-SNAPSHOT.jar \
    com.redis.equities.TsReader
```

`ts.py` creates RedisTimeSeries keys with `TS.CREATE` and loads samples with
`TS.MADD`. `TsReader` reads each series using pipelined `TS.RANGE` calls.

## Runtime Options

Pass these as JVM properties:

| Property | Default | Description |
| --- | ---: | --- |
| `redis.host` | `localhost` | Redis host |
| `redis.port` | `6379` | Redis port |
| `reader.threads` | available processors | Reader thread count |
| `reader.pipelineBatch` | `500` | Number of keys requested per pipeline batch |
| `reader.printPoints` | `false` | Set to `true` to print every decoded point |
| `MAX_KEYS` | `0` | Maximum keys to read; `0` reads all generated keys |

`NUM_SYMBOLS`, `NUM_RISKS`, and `MAX_KEYS` can also be supplied as environment
variables. Environment variables take precedence over the same JVM properties
for these values.

Example with a smaller dataset:

```bash
NUM_SYMBOLS=100 NUM_RISKS=1 python3 ../str.py | redis-cli --pipe

java -DNUM_SYMBOLS=100 \
    -Dredis.host=localhost \
    -Dredis.port=6379 \
    -Dreader.threads=1 \
    -Dreader.pipelineBatch=500 \
    -Dreader.printPoints=false \
    -DMAX_KEYS=0 \
    -cp target/redis-reader-1.0-SNAPSHOT.jar \
    com.redis.equities.RedisReader
```

## Key Format

Both Python loaders and Java readers use the same deterministic key format:

```text
{SYMxxxx}:TENOR:RISK
```

Example:

```text
{SYM0001}:1D:0.01
```

Current defaults:

- `NUM_SYMBOLS=5000`
- `NUM_RISKS=1`
- `TENORS=["1D"]`
- `num_days=5040`

With the defaults, each loader creates `5000` keys and `25,200,000` data
points.
